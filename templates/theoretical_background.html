Expert System (El Ex) for Theorem Proving

Method

This method is called Analytical Tableau, its goal is to find where a given set of statements are satisfiable. If a given set of axioms and inverse of a statement are unsatisfiable, then the statement must be True. This is a proof by contradiction. If the method finds a set of satisfying values for the axioms and the inverse of the statement, then this set becomes a counterexample and therefore a disproof of the statement given the axioms.

How it works

This method works by assuming all the axioms to be True and that the inverse of the given statement is True and then tries to find conflicts. It does this by expanding the given statements in one of 2 ways, non-branching or branching.
An example of non-branching would be if the statement (x*y) is True;therefore, it is known that x is True, and y is True, so the statement (x*y) can be expanded to 2 statements: x, y.
An example of branching would be if the statement (x+y) is True; therefore, it is known that one of x or y is True (or both), so the current tree branches into 2 paths. In one of them, (x+y) is reduced to x, in the other, (x+y) is reduced to y.
A list of the different operations and how they are expanded is stated below:
Non-branching:
  • ~~x reduces to x
  • x*y reduces to: x, y
  • ~(x+y) reduces to: ~x, ~y
  • ~(x>>y) reduces to: x, ~y
Branching:
  • x+y reduces to 2 branches: x, y
  • ~(x*y) reduces to 2 branches: ~x, ~y
  • x>>y reduces to 2 branches: ~x, y
After the given statements are expanded and branched fully, conflicts are searched for. Conflicts occur if both x and ~x are found as statements for any variable x. If a branch is not found with a conflict, then the branch represents a counterexample. If all branches are closed with conflicts, then the statements are not satisfiable; therefore, it is a proof by contradiction.
